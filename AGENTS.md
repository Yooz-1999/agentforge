# AgentForge 协作规则

## 项目记忆

每次在这个仓库工作前，先读取 [skills/agentforge-project/SKILL.md](/Users/zichen/Documents/go/src/agentforge/skills/agentforge-project/SKILL.md:1)。

如果本次对话产生了以后还会用到的信息，更新 [skills/agentforge-project/references/project-memory.md](/Users/zichen/Documents/go/src/agentforge/skills/agentforge-project/references/project-memory.md:1)。

应该写入项目记忆的信息包括：

1. 已确认的本机环境细节
2. 已确定的架构决策
3. 已验证的启动步骤
4. 重复踩坑的问题和解决办法
5. 用户明确表达的项目偏好
6. 主要接口的真实实现状态

不要写入项目记忆的信息包括：

1. 密码、token、API key 等秘密信息
2. 一次性测试数据
3. 没确认过的猜测
4. 大段日志
5. 临时排查过程

## 沟通

默认使用中文回复。

解释问题时先说结论，再说原因和处理办法。

排查问题时按最可能原因到较少见原因排序。
